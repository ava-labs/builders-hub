# P-Chain RPC API

The P-Chain (Platform Chain) manages validators, staking, subnets, and blockchain lifecycle on Avalanche. It exposes JSON-RPC methods for chain info, staking operations, subnet administration, fees, and more.

### Endpoint

- HTTP (JSON-RPC): `/ext/bc/P`

All methods use POST with a JSON-RPC body and a `platform.*` method name.

### Sections

Browse P-Chain RPC methods by topic:

- Chain Info: see `chain-info/`
- Balances & UTXOs: see `balances-&-utxos/`
- Validators: see `validators/`
- Staking: see `staking/`
- Subnets: see `subnets/`
- Blockchains: see `blockchains/`
- Blocks: see `blocks/`
- Transactions: see `transactions/`
- Fees: see `fees/`

Each folder contains individual method pages (e.g., `platform.getHeight`, `platform.getCurrentValidators`, `platform.getSubnets`, etc.).

### Quick examples

Get current height:

```bash
curl -s -X POST \
  -H 'content-type: application/json' \
  --data '{
    "jsonrpc":"2.0",
    "id":1,
    "method":"platform.getHeight",
    "params":{}
  }' \
  https://api.avax.network/ext/bc/P
```

Get subnets by IDs:

```bash
curl -s -X POST \
  -H 'content-type: application/json' \
  --data '{
    "jsonrpc":"2.0",
    "id":1,
    "method":"platform.getSubnets",
    "params": { "ids": [] }
  }' \
  https://api.avax.network/ext/bc/P
```

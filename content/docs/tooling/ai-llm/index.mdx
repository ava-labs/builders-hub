The Builder Hub provides AI-friendly access to documentation through standardized formats. Whether you're building a chatbot, using Claude/ChatGPT, or integrating with AI development tools, we offer multiple ways to access our docs.

## Endpoints Overview

| Endpoint | Purpose |
| :------- | :------ |
| `/llms.txt` | AI sitemap - structured index of all documentation |
| `/llms-full.txt` | Complete docs in one file for full context loading |
| `/api/llms/page?path=...` | Fetch any single page as markdown |
| `/api/mcp` | MCP server for dynamic search and retrieval |

## llms.txt

A structured markdown index following the [llms.txt standard](https://llmstxt.org/). Use this for content discovery.

```
https://build.avax.network/llms.txt
```

Returns organized sections (Documentation, Academy, Integrations, Blog) with links and descriptions.

## llms-full.txt

All documentation content in a single markdown file for one-time context loading.

```
https://build.avax.network/llms-full.txt
```

Contains 1300+ pages. For models with limited context, use the MCP server or individual page endpoint instead.

## Individual Pages

Fetch any page as markdown:

```
https://build.avax.network/api/llms/page?path=/docs/primary-network/overview
https://build.avax.network/api/llms/page?path=/academy/blockchain-fundamentals/blockchain-intro
```

Supports `/docs/`, `/academy/`, `/integrations/`, and `/blog/` paths.

## MCP Server

The [Model Context Protocol](https://modelcontextprotocol.io/) server enables AI systems to search and retrieve documentation dynamically with intelligent search capabilities.

**Endpoint:** `https://build.avax.network/api/mcp`

### Features

- **Synonym-aware search**: Automatically maps Avalanche terminology (subnet↔L1, AWM↔Warp↔ICM, etc.)
- **Code-focused search**: Find code examples filtered by language (Solidity, TypeScript, Go, Bash, etc.)
- **Fuzzy matching**: Handles typos and partial matches
- **Multi-source search**: Searches across docs, academy, integrations, and blog
- **Related content discovery**: Find related pages to explore topics further
- **Section browsing**: Browse all content within a topic area

### Tools

| Tool | Purpose |
| :--- | :------ |
| `avalanche_docs_search` | Search docs by query with synonym expansion and fuzzy matching |
| `avalanche_docs_search_code` | Search specifically for code examples with language filtering |
| `avalanche_docs_fetch` | Get a specific page by URL path |
| `avalanche_docs_list_sections` | List all sections with page counts and descriptions |
| `avalanche_docs_get_related` | Find pages related to a specific page |
| `avalanche_docs_get_section` | Get all pages within a documentation section |

### Claude Code Setup

Add the MCP server to your project with a single command:

```bash
claude mcp add builders-hub --transport http --url https://build.avax.network/api/mcp
```

Or add to your `.claude/settings.json`:

```json
{
  "mcpServers": {
    "builders-hub": {
      "transport": {
        "type": "http",
        "url": "https://build.avax.network/api/mcp"
      }
    }
  }
}
```

### Claude Desktop Setup

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json

    "builders-hub": {
```

### JSON-RPC Protocol

The MCP server uses JSON-RPC 2.0 for communication:

```bash
curl -X POST https://build.avax.network/api/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"avalanche_docs_search","arguments":{"query":"create L1","limit":5}}}'
```

### Search Tool Examples

**Search all documentation:**

The search automatically expands queries with synonyms. For example, searching for "subnet" will also find "L1" content:

```bash
curl -X POST https://build.avax.network/api/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": {
      "name": "avalanche_docs_search",
      "arguments": {
        "query": "smart contracts",
        "limit": 10
      }
    }
  }'
```

**Filter by source:**

```bash
# Search only academy content
curl -X POST https://build.avax.network/api/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/call",
    "params": {
      "name": "avalanche_docs_search",
      "arguments": {
        "query": "blockchain basics",
        "source": "academy",
        "limit": 5
      }
    }
  }'
```

**Filter by section:**

```bash
# Search only cross-chain documentation
curl -X POST https://build.avax.network/api/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",

    "method": "tools/call",
    "params": {
      "name": "avalanche_docs_search",
      "arguments": {
        "query": "messaging",
        "section": "cross-chain",
        "limit": 5
      }
    }
  }'
```

### Avalanche Terminology Support

The MCP server understands common Avalanche terminology and automatically expands searches:

| Term | Also matches |
| :--- | :----------- |
| `l1` | subnet, layer1, blockchain |
| `subnet` | l1, avalanche l1 |
| `icm` | interchain messaging, teleporter, awm, cross-chain |
| `awm` | avalanche warp messaging, warp messages |
| `ictt` | interchain token transfer, token bridge, bridge |
| `cli` | avalanche-cli, command line |
| `precompile` | precompiles, stateful precompile |

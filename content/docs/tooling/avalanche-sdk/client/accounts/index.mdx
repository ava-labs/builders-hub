## Overview

The Avalanche Client SDK provides comprehensive account management capabilities that extend beyond traditional EVM accounts. Unlike standard Ethereum wallets, Avalanche accounts can operate across multiple chains (P-Chain, X-Chain, and C-Chain) with different address formats and signing mechanisms.

**Key Features:**

- Multi-chain account support (P, X, C chains)
- Unified address management
- Multiple account types (Private Key, Mnemonic, HD Key)
- Native Avalanche and EVM compatibility
- Cross-chain transaction signing

## Architecture Overview

Avalanche accounts extend standard Ethereum accounts with XP (X-Chain and P-Chain) capabilities, enabling seamless interaction across all Avalanche chains with a single account.

### Account Structure

```typescript
import type { AvalancheAccount } from "@avalanche-sdk/client/accounts";
import type { Account, Address } from "@avalanche-sdk/client/accounts";

type AvalancheAccount = {
  evmAccount: Account; // For C-Chain operations
  xpAccount?: XPAccount; // For X/P-Chain and C-Chain Atomic operations
  getXPAddress: (chain?: "X" | "P" | "C", hrp?: string) => XPAddress;
  getEVMAddress: () => Address;
};
```

### Quick Start

```typescript
import { privateKeyToAvalancheAccount } from "@avalanche-sdk/client/accounts";
import { createAvalancheWalletClient } from "@avalanche-sdk/client";
import { avalanche } from "@avalanche-sdk/client/chains";
import type { AvalancheAccount } from "@avalanche-sdk/client/accounts";
import { avaxToNanoAvax } from "@avalanche-sdk/client/utils";

// 1. Create account from private key
const account: AvalancheAccount = privateKeyToAvalancheAccount("0x...");

// 2. Create wallet client
const walletClient = createAvalancheWalletClient({
  account,
  chain: avalanche,
  transport: { type: "http" },
});

// 3. Get all addresses
const addresses = {
  evm: account.getEVMAddress(), // C-Chain: 0x742d35Cc...
  xChain: account.getXPAddress("X"), // X-Chain: X-avax1...
  pChain: account.getXPAddress("P"), // P-Chain: P-avax1...
};

// 4. Send transaction
try {
  const pChainExportTxnRequest = await walletClient.pChain.prepareExportTxn({
    exportedOutputs: [
      {
        addresses: [account.getXPAddress("P", "fuji")], // P-fuji19fc97zn3mzmwr827j4d3n45refkksgms4y2yzz
        amount: avaxToNanoAvax(0.00001), // 0.00001 AVAX = 10_000 nAVAX
      },
    ],
    destinationChain: "C",
  });

  const sendTxnResponse = await walletClient.sendXPTransaction(
    pChainExportTxnRequest
  );

  console.log("sendTxnResponse", sendTxnResponse);
} catch (error) {
  console.error("Transaction failed:", error);
}
```

## Account Types

### Local Accounts

Local accounts store signing keys on your machine and perform signing **before** broadcasting transactions. The signing happens in your application's memory or secure storage.

<Callout>
  **Security:** Private keys are stored locally. Never expose them in
  client-side code or commit them to version control.
</Callout>

**Best for:**

- Server-side applications
- Command-line tools
- Automated bots and services
- Applications requiring full programmatic control

**Features:**

- Private keys stored locally
- Signing performed before broadcasting
- Full control over transaction signing
- Support for all Avalanche chains (P, X, C)
- No external wallet dependency

**Account Creation Options:**

- [Private Key Accounts](accounts/local/private-key) - Simple key-based accounts
- [Mnemonic Accounts](accounts/local/mnemonic) - Human-readable recovery phrases
- [HD Key Accounts](accounts/local/hd-key) - Hierarchical deterministic accounts

### JSON-RPC Accounts

JSON-RPC accounts defer signing to external wallets (like MetaMask or Core) over JSON-RPC. Signing happens in the user's browser extension or mobile wallet.

**Best for:**

- Browser-based applications
- User-facing dApps
- Mobile applications
- Applications requiring user approval for transactions

**Features:**

- Signing delegated to external wallet
- Browser wallet integration
- User controls private keys
- Familiar user experience
- No private key exposure in your app

**Learn more:** [JSON-RPC Accounts](accounts/json-rpc)

## Key Components

### EVM Account

Each Avalanche account contains an EVM account for C-Chain operations:

```typescript
import type { Account } from "@avalanche-sdk/client/accounts";

const evmAccount: Account = account.evmAccount;

console.log("Address:", evmAccount.address);
console.log("Type:", evmAccount.type); // "local" | "json-rpc"
console.log("Source:", evmAccount.source); // "privateKey" | "mnemonic" | "hdKey"
```

**Use cases:**

- Smart contract interactions
- C-Chain transactions
- ERC-20 token transfers
- Standard EVM operations

### XP Account

Each Avalanche account contains an XP account for X-Chain and P-Chain operations:

```typescript
import type { LocalXPAccount } from "@avalanche-sdk/client/accounts";

if (account.xpAccount) {
  const xpAccount: LocalXPAccount = account.xpAccount;

  console.log("Public Key:", xpAccount.publicKey);
  console.log("Type:", xpAccount.type); // "local" | "json-rpc"
  console.log("Source:", xpAccount.source); // "privateKey" | "mnemonic" | "hdKey"
}
```

**Example Use cases:**

- UTXO transactions
- X-Chain asset transfers
- P-Chain staking operations
- C-Chain Atomic transactions

## Address Formats

Avalanche supports multiple address formats across its chains. EVM addresses (C-Chain) follow standard Ethereum format, while XP addresses (X-Chain and P-Chain) use Bech32 encoding.

**Address Types:**

- **EVM Addresses:** `0x...` format for C-Chain (Ethereum-compatible)
- **XP Addresses:** `avax1...` format for X/P-Chain (Bech32-encoded)
- **Chain-Prefixed:** `X-avax1...` or `P-avax1...` for explicit chain identification

**Learn more:** [Network-Specific Addresses](accounts/local/addresses)

## Complete Examples

### Example 1: Multi-Chain Balance Checker

```typescript
import { privateKeyToAvalancheAccount } from "@avalanche-sdk/client/accounts";
import { createAvalancheWalletClient } from "@avalanche-sdk/client";
import { avalanche } from "@avalanche-sdk/client/chains";
import type { AvalancheAccount } from "@avalanche-sdk/client/accounts";

async function checkAllChainBalances() {
  const account: AvalancheAccount = privateKeyToAvalancheAccount(
    process.env.PRIVATE_KEY!
  );

  const walletClient = createAvalancheWalletClient({
    account,
    chain: avalanche,
    transport: { type: "http" },
  });

  // Get all addresses
  const addresses = {
    evm: account.getEVMAddress(),
    xChain: account.getXPAddress("X"),
    pChain: account.getXPAddress("P"),
  };

  console.log("Checking balances for:", addresses);

  try {
    // C-Chain balance
    const evmBalance = await walletClient.getBalance({
      address: addresses.evm,
    });
    console.log("C-Chain balance:", evmBalance);

    // X-Chain balance
    const xChainBalance = await walletClient.xChain.getBalance({
      addresses: [addresses.xChain],
    });
    console.log("X-Chain balance:", xChainBalance);

    // P-Chain balance
    const pChainBalance = await walletClient.pChain.getBalance({
      addresses: [addresses.pChain],
    });
    console.log("P-Chain balance:", pChainBalance);

    return { evmBalance, xChainBalance, pChainBalance };
  } catch (error) {
    console.error("Failed to check balances:", error);
    throw error;
  }
}
```

### Example 2: Cross-Chain Transfer

```typescript
import { privateKeyToAvalancheAccount } from "@avalanche-sdk/client/accounts";
import { createAvalancheWalletClient } from "@avalanche-sdk/client";
import { avalanche } from "@avalanche-sdk/client/chains";
import type { AvalancheAccount } from "@avalanche-sdk/client/accounts";

async function transferBetweenChains(
  sourceChain: "P" | "X" | "C",
  amount: number
) {
  const account: AvalancheAccount = privateKeyToAvalancheAccount(
    process.env.PRIVATE_KEY!
  );

  const walletClient = createAvalancheWalletClient({
    account,
    chain: avalanche,
    transport: { type: "http" },
  });

  try {
    // Creating a export transaction request from the C-chain to the P-chain
    const cChainExportTxnRequest = await walletClient.cChain.prepareExportTxn({
      destinationChain: "P",
      fromAddress: account.getEVMAddress(), // 0x76Dd3d7b2f635c2547B861e55aE8A374E587742D
      exportedOutput: {
        addresses: [account.getXPAddress("P", "fuji")], // P-fuji19fc97zn3mzmwr827j4d3n45refkksgms4y2yzz
        amount: avaxToNanoAvax(0.0001), // 0.0001 AVAX = 100_000 nAVAX
      },
    });

    // Signing and sending the export transaction request to the C-chain
    const sendTxnResponse = await walletClient.sendXPTransaction(
      cChainExportTxnRequest
    );

    // Waiting for the export transaction to be confirmed on the C-chain
    await walletClient.waitForTxn(sendTxnResponse);

    // Logging the export transaction response
    console.log(
      "Avax exported to Atomic Memory for C-chain to P-chain transfer",
      sendTxnResponse
    );

    // Creating a import transaction request in P-chain
    const pChainImportTxnRequest = await walletClient.pChain.prepareImportTxn({
      sourceChain: "C",
      importedOutput: {
        addresses: [account.getXPAddress("P", "fuji")], // P-fuji19fc97zn3mzmwr827j4d3n45refkksgms4y2yzz
      },
    });

    // Signing and sending the import transaction request to the P-chain
    const sendTxnResponse2 = await walletClient.sendXPTransaction(
      pChainImportTxnRequest
    );

    // Waiting for the import transaction to be confirmed on the P-chain
    await walletClient.waitForTxn(sendTxnResponse2);

    // Logging the import transaction response
    console.log(
      "Avax imported from Atomic Memory from C-chain to P-chain transfer",
      sendTxnResponse2
    );
  } catch (error) {
    console.error("Cross-chain transfer failed:", error);
    throw error;
  }
}
```

## Security Best Practices

### Private Key Management

<Callout>
  **Never expose private keys in client-side code or commit them to version
  control.**
</Callout>

```typescript
// ✅ Good: Use environment variables
const account = privateKeyToAvalancheAccount(process.env.PRIVATE_KEY!);

// ❌ Bad: Hardcoded private key
const account = privateKeyToAvalancheAccount("0x1234...");
```

### Mnemonic Security

<Callout>
  **Never expose mnemonics in client-side code or commit them to version
  control.**
</Callout>

```typescript
// ✅ Good: Use environment variables
const account = mnemonicsToAvalancheAccount(process.env.MNEMONIC!);

// ❌ Bad: Hardcoded mnemonic
const account = mnemonicsToAvalancheAccount("abandon abandon...");
```

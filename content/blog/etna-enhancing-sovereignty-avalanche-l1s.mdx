The Avalanche Network released upgrade code for "Etna," activated on Mainnet at 12 PM ET (5 PM UTC) on December 16, 2024. This transformative upgrade incorporates six Avalanche Community Proposals (ACPs) that fundamentally reshape how L1 networks operate on Avalanche.

**Production Release**: AvalancheGo v1.12.0
**Mainnet Activation**: December 16, 2024 at 12:00 PM ET
**Testnet Activation**: Successfully tested on Fuji on November 25, 2024

## Avalanche Community Proposals Included

The Etna upgrade incorporates six major ACPs:

- **ACP-77**: Reinventing Subnets as Sovereign L1 Networks
- **ACP-103**: Dynamic Fees for P-Chain
- **ACP-118**: Warp Signature Interface Standardization
- **ACP-125**: Reduced C-Chain Base Fees
- **ACP-131**: Cancun EIPs Activation
- **ACP-151**: P-Chain Height Context

## Sovereign L1 Networks: A Paradigm Shift

### Over 99.9% Cost Reduction

ACP-77 introduces mechanisms for creating sovereign Layer 1 blockchains with lightweight, cost-efficient validation. Instead of requiring 2,000 AVAX stakes, L1 validators pay dynamic continuous fees of **approximately 1.3 AVAX per month**.

This represents **over 99.9% reduction in upfront costs** for blockchain creation.

### No Primary Network Synchronization Required

Validators aren't required to sync the Primary Network, unlike Subnet validators. This change:

- **Reduces Resource Requirements**: Lower storage and bandwidth needs
- **Faster Onboarding**: Validators can start more quickly
- **True Sovereignty**: L1s operate independently of Primary Network state

### Custom Validator Management

The upgrade enables custom validator management through smart contracts. Projects can:

- Use provided **proof-of-authority** contracts for permissioned networks
- Deploy **proof-of-stake** contracts for decentralized validation
- Create **custom validation logic** for unique use cases

This flexibility allows L1 networks to implement governance models that match their specific requirements.

## C-Chain Fees Reduced by 96%

### Minimum Base Fee Reduction

### Impact on Users

This change lowers the EIP-1559 fee floor, making transactions significantly cheaper during low network congestion periods. The base fee adjusts dynamically based on demand, while this update establishes a lower minimum threshold.

**Benefits**:
- **Lower Transaction Costs**: Minimal fees during low-activity periods
- **Better UX**: More affordable for high-frequency operations
- **Competitive Positioning**: Attractive fees for developers and users

## Dynamic Fees on P-Chain

### From Fixed to Dynamic

Following ACP-103's specification, dynamic fees now apply to the P-Chain. Previously, fixed fees provided simplicity but didn't account for congestion.

### New Fee Mechanism

The new mechanism:
- **Reduces transaction costs** during normal operation
- **Improves resilience** against load spikes
- **Prepares infrastructure** for increased usage from ACP-77's new validator management capabilities

This adaptive pricing ensures the P-Chain remains efficient and economical while handling variable demand.

## Interchain Messaging Standards

### VM-Agnostic Signature Interface

ACP-118 establishes a VM-agnostic standard for requesting signatures on Avalanche Interchain Messages (ICM).

### Simplifying Cross-Chain Development

Valid signatures require querying a blockchain's validator set to request threshold-majority signature aggregation. This standardization:

- **Centralizes Logic**: Signature logic moves to AvalancheGo instead of individual VM codecs
- **Simplifies Implementation**: Easier to build signature aggregators
- **Improves Consistency**: Standard interface across all VMs

## Cancun EIPs Activation

### Ethereum Compatibility

ACP-131 activates Cancun Ethereum Improvement Proposals on the C-Chain, ensuring continued compatibility with the latest Ethereum standards.

**Key Improvements**:
- **Proto-danksharding (EIP-4844)**: Blob transactions for data availability
- **Transient Storage (EIP-1153)**: Gas-efficient temporary storage
- **Additional Opcodes**: Enhanced smart contract capabilities

## P-Chain Height Context

### Enhanced Context Awareness

ACP-151 provides P-Chain height context to validators, enabling:

- **Time-Based Logic**: Smart contracts can reference P-Chain block heights
- **Coordination**: Better synchronization across chains
- **Advanced Features**: Foundation for time-locked operations and scheduled upgrades

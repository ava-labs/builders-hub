# Launch a Token on Avalanche

Create, deploy, and manage your own ERC-20 token on Avalanche. This guide covers everything from token contract development to adding liquidity and distribution.

## What You'll Create

By the end of this guide, you'll have:
- A custom ERC-20 token deployed on Avalanche
- Understanding of token economics and distribution
- Knowledge of how to add liquidity and list your token

## Prerequisites

- Basic understanding of blockchain and tokens
- MetaMask or Core wallet setup
- Some AVAX for gas fees (get from [faucet](https://faucet.avax.network/) for testnet)

## Step 1: Design Your Token

### Token Planning Checklist

Before creating your token, define:
- **Name**: The full name (e.g., "My Awesome Token")
- **Symbol**: 3-5 character ticker (e.g., "MAT")
- **Total Supply**: How many tokens will exist
- **Decimals**: Usually 18 for ERC-20 tokens
- **Use Case**: What problem does your token solve?

### Token Economics (Tokenomics)

Consider your token distribution:
- Team allocation (typically 10-20%)
- Community rewards (20-40%)
- Liquidity provision (10-20%)
- Treasury/Development (10-30%)
- Public sale (remaining percentage)

## Step 2: Create Your Token Contract

### Option 1: Use OpenZeppelin (Recommended)

Create a secure token using battle-tested contracts:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Burnable.sol";

contract MyToken is ERC20, Ownable, ERC20Burnable {
    constructor(
        string memory name,
        string memory symbol,
        uint256 totalSupply,
        address owner
    ) ERC20(name, symbol) {
        _mint(owner, totalSupply * 10**decimals());
        _transferOwnership(owner);
    }
    
    function mint(address to, uint256 amount) public onlyOwner {
        _mint(to, amount);
    }
}
```

## Step 3: Deploy Your Token

### Set Up Development Environment

```bash
npm install -g @remix-project/remixd
# Or use Hardhat for local development
npm install --save-dev hardhat
```

### Deploy with Remix (Easiest)

1. Go to [Remix IDE](https://remix.ethereum.org/)
2. Create new file: `MyToken.sol`
3. Paste your token contract code
4. Install OpenZeppelin:
   ```
   npm install @openzeppelin/contracts
   ```
5. Compile your contract
6. Connect MetaMask to Avalanche Fuji Testnet
7. Deploy with parameters:
   - Name: "My Awesome Token"
   - Symbol: "MAT"
   - Total Supply: 1000000 (will be 1M tokens with 18 decimals)
   - Owner: Your wallet address

### Deploy with Hardhat

Create `scripts/deploy-token.js`:

```javascript
async function main() {
  const [deployer] = await ethers.getSigners();
  
  console.log("Deploying token with account:", deployer.address);
  
  const MyToken = await ethers.getContractFactory("MyToken");
  const token = await MyToken.deploy(
    "My Awesome Token",  // name
    "MAT",              // symbol
    1000000,            // total supply (before decimals)
    deployer.address    // owner
  );
  
  await token.deployed();
  
  console.log("Token deployed to:", token.address);
  console.log("Total supply:", await token.totalSupply());
}

main().catch((error) => {
  console.error(error);
  process.exitCode = 1;
});
```

Deploy:
```bash
npx hardhat run scripts/deploy-token.js --network fuji
```

## Step 7: Token Distribution

### Airdrop Script

```javascript
// Simple airdrop script
async function airdrop() {
  const token = await ethers.getContractAt("MyToken", TOKEN_ADDRESS);
  
  const recipients = [
    "0x1234...", // address 1
    "0x5678...", // address 2
    // ... more addresses
  ];
  
  const amount = ethers.utils.parseEther("100"); // 100 tokens each
  
  for (const recipient of recipients) {
    await token.transfer(recipient, amount);
    console.log(`Sent 100 tokens to ${recipient}`);
  }
}
```

## Legal Considerations

⚠️ **Important**: Token creation may have legal implications:
- Consult legal experts before launching
- Understand securities law in your jurisdiction
- Consider regulatory compliance
- Implement proper KYC/AML if required

## Common Pitfalls

### Avoid These Mistakes

1. **Unlimited minting** without proper controls
2. **No burn mechanism** leading to inflation
3. **Poor tokenomics** that don't incentivize holding
4. **No utility** beyond speculation
5. **Inadequate security** audits
6. **Rushing to mainnet** without thorough testing
